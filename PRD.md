# PRD: Сбор заявок на планирование бюджета 2027 год

**Версия:** 1.0
**Дата:** 25.06.2026
**Статус:** для согласования с руководством
**Владелец продукта:** Управление PR

---

## 1. Назначение и контекст

Управление PR ведёт более 10 проектов параллельно. Текущий процесс взаимодействия со смежными отделами и регионами построен на разрозненной переписке в почте и чатах, из-за чего:

- статусы проектов не видны заинтересованным сторонам без точечных запросов;
- заявки и согласования теряются между каналами связи;
- причины задержек неочевидны и требуют ручного уточнения;
- руководство не имеет сводной картины по всем активным проектам и заявленным бюджетам, в том числе из регионов.

Продукт создаётся для приёма и отслеживания заявок на планирование PR-бюджета 2027 года, включая заявки из регионов, с прозрачными статусами и понятной ответственностью по каждой заявке.

## 2. Цели продукта

1. Создать единую точку приёма заявок на PR-проекты и бюджет, включая заявки из регионов.
2. Обеспечить прозрачность статусов всех заявок для всех участников процесса.
3. Зафиксировать бюджет и регион по каждой заявке для последующей консолидации бюджета 2027 года.
4. Снизить количество коммуникаций «уточнение статуса» за счёт самообслуживания (доступ на просмотр).
5. Дать руководству сводную аналитику по количеству заявок, бюджету и просрочкам.

## 3. Пользователи и роли

| Роль | Доступ | Возможности |
|---|---|---|
| PR-менеджер | Полный доступ | Создание и ведение заявок, изменение статусов, назначение ответственных, указание блокеров |
| Руководитель PR-отдела | Полный доступ + аналитика | Все функции PR-менеджера, сводная аналитика, фильтры по просрочкам/блокерам |
| Представитель смежного отдела / региона | Просмотр + создание заявки | Подача заявки на проект и бюджет через форму, просмотр статуса своих заявок |
| Руководство компании | Просмотр (read-only) | Сводный дашборд по всем заявкам, суммарный бюджет, ключевые метрики |

## 4. Функциональные требования

### 4.1 Приём заявок
- Форма заявки на новый PR-проект/бюджет со следующими полями:
  - **Название проекта** — свободный текст;
  - **Тип проекта** — выбор из списка (Запуск продукта / Кризисная коммуникация / Ивент / Публикация в СМИ) **или** ручной ввод собственного типа;
  - **Отдел-заказчик** — свободный текст;
  - **Цель и ключевые сообщения** — свободный текст (бриф);
  - **Контактное лицо заказчика** — свободный текст;
  - **Дедлайн** — дата;
  - **Регион** — выбор из списка (Алматы / Астана / Шымкент / Караганда / Актобе) **или** ручной ввод другого региона;
  - **Бюджет, ₸** — числовое значение.
- Заявка не попадает в работу без заполненного брифа (обязательные поля).
- Автоматическое уведомление PR-команды о новой заявке *(этап 2)*.

### 4.2 Управление заявками
- Канбан-доска со статусами: Новая заявка → Бриф согласован → В работе → На согласовании → Готово к публикации → Опубликовано → Мониторинг результатов.
- Карточка заявки: название, тип, регион, бюджет, ответственный PR-менеджер, контактное лицо, дедлайн, поле «Блокер» (причина задержки).
- История изменений статусов по каждой заявке.
- Возможность редактирования брифа, ответственного, дедлайна, региона, бюджета и блокера для ролей PR-менеджер/руководитель.

### 4.3 Доступ и видимость
- Смежные отделы и регионы видят только свои заявки, без права редактирования.
- Сводный дашборд для руководства: общее количество заявок, суммарный бюджет по регионам, количество заявок с блокером, количество просроченных.
- Фильтры: по типу проекта, отделу-заказчику, региону, наличию блокера, просрочке.

### 4.4 Уведомления *(этап 2)*
- Уведомления при смене статуса заявки.
- Эскалация ответственному руководителю при превышении SLA на одной стадии.
- Напоминания о приближающихся дедлайнах.

### 4.5 Аналитика и отчётность *(этап 2)*
- Отчёт по суммарному запрошенному бюджету 2027 года в разрезе регионов и отделов.
- Отчёт по среднему времени прохождения заявки по стадиям.
- Экспорт отчётов в PDF/Excel.

### 4.6 Администрирование *(этап 2)*
- Управление пользователями и ролями.
- Настройка SLA по типам проектов.
- Настройка контактных лиц по отделам и регионам.

## 5. Нефункциональные требования

| Параметр | Требование |
|---|---|
| Платформы | Веб-приложение (адаптивная вёрстка) + мобильные приложения iOS и Android *(этап 3)* |
| Производительность | Загрузка дашборда не более 2 секунд при 100+ активных заявках |
| Безопасность | Ролевой доступ (RBAC), журналирование действий пользователей |
| Локализация | Русский язык интерфейса (приоритет) |
| Масштабируемость | Поддержка роста числа заявок и пользователей без изменения архитектуры |

## 6. Метрики успеха

- 100% заявок на бюджет 2027 года ведутся через систему, а не в почте/чатах, через 1 месяц после запуска.
- Снижение количества обращений «какой статус у заявки» по сравнению с текущим процессом.
- Полная и консолидированная картина запрошенного бюджета по всем регионам к моменту защиты бюджета.

## 7. Текущий статус разработки

На текущем этапе создан **интерактивный прототип (MVP) веб-версии** — `PR-Tracker.html` — демонстрирующий:
- канбан-доску по всем статусам;
- форму подачи заявки с регионом и бюджетом, включая ручной ввод произвольного типа проекта и региона;
- ролевой доступ (PR-менеджер, руководитель, смежный отдел/регион, руководство);
- фильтры и сводную панель метрик, включая суммарный бюджет.

Прототип предназначен для демонстрации логики и UX перед запуском полноценной разработки (backend, мобильные приложения, уведомления, аналитика — см. разделы 4.4–4.6).
